# time-journal 开发计划

> 当前目标：不要继续加大功能面，而是降低记录成本、强化“计划 vs 实际”的对照感，并统一视觉系统。

核心判断：App 底子成立，方向是暖色、克制、手账感、低压力。下一步不是推翻，而是把它从“纸质手账电子化”优化成“真正适合手机使用的数字手账”。

产品一句话：

> 帮用户看清：我原本打算怎么过，最后实际怎么过。

执行原则：

1. 少填：能自动生成的，不让用户重复写。
2. 少点：高频操作不超过两步。
3. 对照：计划和实际必须放在一起看。
4. 克制：不要过度游戏化，不要过度统计化。
5. 温和：不评价用户，只帮助用户看清自己。

## 2. P0：最优先改，直接影响体验

### P0-1 首页合并“计划完成”和“实际完成”为“今日对照”

**问题**

计划完成里写一次，实际完成里又写一次，用户会觉得重复。

**目标**

把计划和实际放进同一个时间块中展示，形成“今日对照”或“今日时间线”。

**推荐结构**

```text
今日对照
看看计划和现实差在哪里

09:00 - 10:00
计划：睡觉
实际：睡觉
状态：一致

10:00 - 11:00
计划：学习
实际：未记录
状态：待补
```

**范围**

- 主要改 `features/journal/`
- 可读 `app/lib/app/theme.dart`、`app/lib/app/copy.dart`
- 默认不改 Drift schema；若必须改，先报告 migration 方案

**验收**

- 首页不再显示两个完全独立的大区块“计划完成 / 实际完成”。
- 同一时间块可同时展示计划与实际。
- 状态至少能表达：一致、待补、偏差、有新增实际。

---

### P0-2 计划卡片增加“按计划完成 / 实际有变”

**问题**

用户完成计划后仍要手动再写一次实际记录。

**目标**

每个计划时间块下方加两个轻按钮：

```text
09:00 - 10:00
睡觉

[按计划完成] [实际有变]
```

点击“按计划完成”后自动生成实际记录；点击“实际有变”后进入实际记录编辑。

**范围**

- `features/journal/`
- repository 层如已有生成实际记录能力，可复用
- 不直接清空或重建数据库

**验收**

- “按计划完成”能继承计划时间和内容生成实际记录。
- “实际有变”能保存与计划不同的实际时间或内容。
- 重复点击不会产生明显重复脏数据，至少要有 UI 层防重复或 repository 层幂等处理。

---

### P0-3 专注结束后自动写入实际记录

**问题**

专注完成后如果只停留在计时结束，手账闭环没有打通。

**目标**

专注完成后弹出完成反馈：

```text
完成一段专注
25 分钟 · 写作业

[记入今天] [补一句备注]
```

点击“记入今天”后，自动加入首页实际记录。

**范围**

- `features/pomodoro/`
- `features/journal/` 的 repository / model
- 可能涉及 copy 文案
- 不做复杂成就系统

**验收**

- 从待办发起专注时，完成记录能关联回该待办。
- 临时任务专注完成后也能写入实际记录。
- 写入后今日统计能更新。

---

### P0-4 首页减少卡片数量和滚动长度

**问题**

首页像把数据库记录都展示出来，信息太满。

**目标**

首页只保留四层：

```text
日期 + 今日状态
待办事项
今日对照
今日统计 / 今天一句话
```

**说明**

“学习时间”改为小统计卡，不要占据过大视觉权重。

示例：

```text
今日统计
计划学习 2 小时
实际学习 3 小时
专注 25 分钟
```

**验收**

- 首页首屏压迫感下降。
- 待办过多时折叠。
- 今日统计单位清晰，如 120 分钟 / 2 小时。

## 3. P1：UI 精致度和模块体验

### P1-1 统一卡片系统

建议固定规则：

```text
页面背景：暖米白
卡片背景：接近白色的奶油色
圆角：16
边框：浅米灰 1px
阴影：弱化，甚至不用明显阴影
页面左右边距：16
卡片内边距：14-16
卡片间距：12-16
```

整体应该更像“纸张层叠”，不要像普通后台管理系统。

---

### P1-2 统一标题前的小红竖线

小红竖线有品牌感，建议保留并组件化：

```text
宽度：3
高度：16
圆角：2
颜色：红棕主色
标题字号：16-18
标题字重：600 / 700
```

所有模块统一使用同一标题组件。

---

### P1-3 红棕色分层使用

红棕色不要承担太多功能。

建议分层：

```text
主操作按钮：红棕色
选中态：浅红背景 + 红棕文字
普通标签：米灰背景 + 深灰文字
睡眠模块：低饱和蓝
危险/删除：单独设计，不复用主红棕
```

---

### P1-4 字体层级统一

建议字号：

```text
页面标题：24 / 700
模块标题：17 / 700
正文：15-16
副文案：13
时间标签：20-24 / 700
专注倒计时：72-88
按钮文字：16 / 600
```

首页里的时间不要每张卡都过大，否则会抢走任务内容注意力。

---

### P1-5 待办事项区简化

当前待办如果同时有 checkbox、左图标、右图标、拖拽符号，会显得重。

建议简化为：

```text
□ 睡觉
□ 写一件小事也可以
□ 吃饭
```

右侧最多保留一个操作：“安排”。

点击待办后弹出底部面板：

```text
安排到时间段
开始专注
标为完成
删除
```

待办超过 5 条自动折叠：

```text
默认显示前 5 条
查看更多 3 条
```

---

### P1-6 专注页强化“任务感”

专注页不要只是番茄钟，要先问“专注哪一件事”。

推荐结构：

```text
专注

今天专注哪一件事？
[ 输入框 ]

从今日待办中选择
[ 睡觉 ] [ 吃饭 ]

25:00

[5分钟] [15分钟]
[25分钟] [45分钟]

[开始专注 25 分钟]
```

专注进行页：

```text
专心当下

睡觉

25:00

需要先停一下？长按确认
```

---

### P1-7 睡眠页调整模块顺序

睡眠页应该围绕一个主操作：

```text
我准备去睡了
```

推荐顺序：

```text
睡眠

今晚目标
23:00 睡 / 07:00 起

[我准备去睡了]

白噪音
雨声 / 海浪 / 篝火 / 风声

今日记录
就寝：未记录
起床：未记录
```

蓝色卡片降饱和：低饱和灰蓝、雾蓝、深奶蓝。不要商务蓝，不要太亮。

白噪音增加：选中态、播放状态、音量入口、停止按钮。

---

### P1-8 我的页改成设置列表

当前“我的”页不要像说明书。

推荐结构：

```text
我的

一周手账
这一周，你是怎么过的      >

数据存储
所有记录都保存在本机      >

关于
时间管理手账 v1.0.0       >

设置
主题 / 导出 / 清空数据     >
```

每一项用统一行组件：左图标 + 标题 + 副标题 + 右箭头。

---

### P1-9 一周手账页重做图表

当前图表如果像占位图，应改成节奏热力图：

```text
这一周的节奏

        一  二  三  四  五  六  日
手账    □  ■  □  □  □  □  □
专注    □  □  □  □  □  □  □
早睡    □  □  □  □  □  □  □
```

用户一眼能看到：哪天记录了、哪天断了、哪类习惯比较稳定。

## 4. P2：交互效率优化

### P2-1 当前时间段突出显示

首页可根据当前时间自动突出当前卡片。

示例：

```text
现在
18:30 - 19:00
计划：吃饭
```

如果当前没有计划：

```text
当前没有计划
要不要补记刚刚这一段？
```

---

### P2-2 自动补记录入口

很多人晚上才补记录，所以首页可以加轻入口：

```text
补记刚刚这一段
```

或：

```text
这段时间，你在做什么？
```

这比让用户主动点“+ 时段”更自然。

---

### P2-3 时间选择器换成滚轮

Material 表盘选择器标准但不一定高效。手账感 App 更适合滚轮选择器：

```text
开始时间：09 : 00
结束时间：10 : 00
```

---

### P2-4 空状态和文案优化

不要只显示：

```text
完成 0 段专注
睡眠 0 天
```

改为：

```text
本周还没有专注记录
挑一个安静的时间开始就好
```

```text
这周还没记录睡眠
今晚开始也不晚
```

```text
这周只写了 1 天
能留下痕迹就已经不错
```

保留这句：

```text
不是「你这周表现如何」，而是「这一周，你是怎么过的」
```

以及：

```text
数字只是镜子，帮你看清自己；怎么调整，只有你知道。
```

## 5. P3：后续可以做，但别急

### P3-1 白噪音轻微动效

可做，但必须克制：极慢背景呼吸、淡雨滴纹理、轻微星点、柔和波纹。

不要强动画，不然睡眠页会变吵。

### P3-2 拖拽安排任务

后续可做：长按待办，拖到时间轴，生成计划。

但不是 MVP 必需。先把点击安排做好，再考虑拖拽。

### P3-3 专注完成小动效

可以有，但不要游戏化。

不要：

```text
获得专注能量
升级
连续打卡奖励
```

建议：

```text
这一段守住了
25 分钟已记入今天
```

### P3-4 数据导出

后续“我的”页可以加：

```text
导出为 Markdown
导出为 CSV
清空本地数据
备份数据
```

## 6. 暂时不建议做

这些会让产品变重：

```text
复杂标签系统
任务优先级矩阵
四象限管理
积分等级系统
复杂成就系统
社交分享
排行榜
过多统计图表
AI 自动评价你今天好不好
```

产品核心不是督促用户变强，而是帮用户看清自己。

不要做成：

```text
效率工具
打卡工具
绩效工具
任务管理器
```

它应该更像：

```text
低压力时间手账
自我观察工具
计划与现实对照本
```
